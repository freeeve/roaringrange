# Task 050 — Go RRVI vector (`.rrvi`) IVFPQ trainer + RRVR rerank — DONE

Done 2026-06-23. The trainer was built as a standalone library, **`~/go-ivfpq`**
(`github.com/freeeve/go-ivfpq`, MIT, tagged `v0.1.1`) — the `fst-go` / `go-stemmers`
spin-out pattern, because the k-means/PQ/IVF guts are reusable and married to nothing
roaringrange-specific; only the `.rrvi`/`.rrvr` serialization is, and that lives here.

## What shipped

- **`~/go-ivfpq`** ([[go-ivfpq-repo]]): `Rng` (xorshift64\*), `KMeans` (Lloyd's),
  `normalize`, `Train` (= `build_ivfpq`), `FromParts` (= `build_ivfpq_from_parts`),
  `Reconstruct`. Pure Go, no cgo / no FAISS.
- **roaringrange `go/vector.go`**: `WriteRRVI(model *ivfpq.Model)` + `WriteRerank` — the
  byte-exact `.rrvi` / `.rrvr` serializers (`go/` now depends on `go-ivfpq v0.1.0`).

## Conformance (two-tier, by design)

Cross-impl reference fixtures are generated by `rust/src/vector_build.rs`
`mod conformance_ref` (gated `RR_UPDATE_FIXTURES=1`, no public-API change). Robust parts
are bit-for-bit (Rng sequence, k-means centroids, the trainer's coarse centroids + list
membership, and the whole deterministic `WriteRRVI`/`WriteRerank` path via golden tests).
PQ codebooks/codes are FMA-fragile across languages, so PQ quality is recall — `recall@10
= 0.85` vs brute force.

## Perf pass (requested: allocations first, then profiling)

- **Allocations: 2762 → 48 /op** in `Train` (deterministic, the reliable win):
  `appendNormalized` writes into the points buffer (no per-vector temp), and a CSR-style
  scatter gives each inverted list an exact-capacity view into two shared backing arrays
  (no growth reallocs). Remaining 48 are fixed-count, not proportional to N.
- **Profiling: `nearest` is ~95% of build time.** Bounds-check-eliminated its squared-L2
  inner loop (≈15–25% on the hot path by an A/B under one thermal window; full-Train
  wall-clock on this laptop is dominated by thermal drift, so the per-loop A/B is the
  trustworthy signal). Output unchanged — all conformance + recall tests pass.
