… works all static on S3 · range-fetched
runs the intersection in a Lambda and returns just this page's result IDs plus facet counts — KB instead of MB for huge result sets, at the cost of the in-region exact-total scan (slow for very common terms)
embeds the query with EmbeddingGemma-300M in a Lambda — a full contextual transformer, for markedly more accurate semantic matches — then range-searches its Gemma-embedded vector index. The default in-browser embedder is model2vec (potion-retrieval-32M): a static distillation that mean-pools precomputed token vectors in wasm — instant and server-free, but lower fidelity.
swaps the monolithic RRS for the RRSS split set — many byte-capped immutable splits, queried with tier/Bloom/facet pruning, booted from one .rrhc bundle
reranks the most-cited matches by lexical relevance via the .rrb impact sidecar — one precomputed tf/length byte per (word, doc), addressed through the posting bitmaps the search already fetched, scored Σ idf × impact. Without it, order is citation rank alone.
